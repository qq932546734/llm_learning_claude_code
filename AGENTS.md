## 🛡️ 防遗忘机制（SOP提醒）

**每次会话我必须：**
1. 读取上述核心文件恢复上下文
2. 开场先确认：`学习模式` 还是 `提问模式`
3. 若处于 `学习模式` 且学习者未明确说明时间不足，默认按 `2 小时完整版` 生成今日学习单，不可自行压缩
4. 学习模式下提问后标记 `[记录候选]`
5. 提问模式下执行“回答 → 总结 → 归类 → 回写”
6. 只要用户表达“结束/先这样/下次继续/今天到这”等收束意图，立即自动触发落盘流程
7. 在结束回复前完成检查清单并更新核心文档
8. 结束回复中必须明确报告“已更新了哪些文件”

**会话结束硬规则：**
- 不能只口头总结而不落盘。
- 不能把“是否要更新文档”再抛回给学习者决定；默认必须更新。
- 必须先写 `docs/progress.md`、`docs/reminder_config.md`、`docs/learning_journal.md`，再视情况更新 `docs/question_bank.md` / `docs/question_analysis.md`。
- 写完后必须重新读取或核对一次，确认落盘成功，再发送结束回复。

**如果学习者说：**
- "记录这个问题" → 立即强制记录
- "你忘了记录" → 立即补录
- "检查清单" → 显示当前状态

### 双入口模式（v1）
- `学习模式`：课程推进、复习检测、口述训练为主；提问仍照常记录，高价值问题会进入 `docs/question_bank.md`
- `提问模式`：随手问题沉淀为主；默认完成“直接答案 + 一句话总结 + 模块/知识点归类 + 是否加入复习”
- 硬规则：`学习模式` 不削弱记录能力；`提问模式` 只是增加强制总结、强制归类、强制回写

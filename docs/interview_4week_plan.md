# 4 周冲刺推进表（主线计划，目标：面试可用）

> 适用起点：2026-03-16
> 学习目标：4 周内完成高频主线补齐、薄弱点加固、口述与变体题训练，达到“面试可用”。
> 默认节奏：每周学习 6 天，1 天弹性补位或休整；按天推进主线，实际单次学习时长不固定。

---

## 总体策略

- 前 2 周：补齐高频主线 + 修复当前薄弱点。
- 第 3 周：完成中高频模块首轮学习，建立系统地图。
- 第 4 周：做面试化收束，集中口述、迁移题、估算题和错题回访。

## 每日标准模板（默认）

- `主线必做`
  - 当天计划日对应的新知识或主线推进任务
- `到期复习`
  - 根据 `docs/reminder_config.md` 与最近学习日期计算出的到期 / 逾期复习
- `收束输出`
  - 3 句总结 / 1 个易错点 / 1 个迁移题 / 更新下次任务

> 使用规则：本文件负责“按天推进的主线顺序”；每日具体学习单需再与 `docs/reminder_config.md`、`docs/progress.md` 合并生成。若某天未完成，不跳过，顺延到最近未完成的计划日。

## 判断是否达到“面试可用”

满足以下大部分条件，即可视为达到当前阶段目标：

- 能在 `2-3 分钟` 内口述高频模块主线，不明显卡壳。
- 面对追问时，能说出“为什么”，而不是只背定义。
- 能完成基础估算题：embedding 参数量、LoRA 显存直觉、prefill/decode 成本拆分。
- 能解释常见工程权衡：tokenizer、解码参数、KV cache、ZeRO/FSDP。
- 能区分“模型能力”与“系统/训练/解码策略表现”。

## Week 4+ 可选延长补强（`E01`–`E06`，默认不插入当前主线）

> 触发条件：仅当 `Week 1 ~ Week 4` 主线完成，或用户明确允许延长学习周期时启用。
> 保护规则：不回头改写当前 `Week 3 / Week 4` 的日程；既有复习队列优先级保持不变；新增主题在正式开始前不提前生成新的间隔复习压力。

### 补强总原则

- 第一层先做“课程资产回写”：
  - 把已学深度从 `docs/learning_journal.md` / `docs/question_analysis.md` 反写回 `modules/`
  - 优先回写 `M09 / M13 / M14 / M15 / M16`
  - 这类工作默认不占主线学习时间
- 第二层再做“新主题扩展”：
  - 新主题只放到延长期或 `Day 7` 弹性日
  - 仍然遵守“当天主线 + 到期复习优先”的合并规则

### 延长期优先级

1. `E01` 评估与基准测试（`benchmark / LLM-as-a-judge`）
2. `E02` 安全与红队（`red teaming / jailbreak / prompt injection`）
3. `E03` 数据工程（`清洗 / 去重 / 配比`）
4. `E04` RAG 与检索（`RAG / retrieval`）
5. `E05` 量化与服务栈（`quantization / vLLM / TGI / TensorRT-LLM`）
6. `E06` 现代架构扩展（`MoE / SSM / Mamba`）

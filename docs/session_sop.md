# 学习会话标准操作流程 (SOP)

> 确保学习连续性、记录完整性的检查清单

---

## 会话开始流程

### 1. 恢复上下文（必须）
```
□ 读取 docs/progress.md
□ 读取 docs/reminder_config.md
□ 读取 docs/question_bank.md（最近沉淀的问题）
□ 读取 docs/question_analysis.md（最后部分）
□ 读取 docs/taxonomy.md（模块编号口径）
□ 检查上次学习日期，判断是否需要复习检测
```

### 2. 模式入口（新增，必须）
```
□ 开场先确认："今天进入学习模式还是提问模式？"
□ 若用户未明确选择：
   - 默认按学习模式开场
   - 若用户一上来就是独立问题，按提问模式响应
□ 明确本轮目标：
   - 学习模式 = 推进课程 / 复习 / 口述训练
   - 提问模式 = 回答问题 / 总结 / 归类 / 回写 / 加入复习
```

### 3. 状态确认
```
□ 向用户确认当前状态："上次我们学到XX，今天继续？"
□ 列出待办事项（如有）："你还有X个费曼输出/练习题待完成"
□ 学习模式提供3个选项：继续/复习/新主题
□ 提问模式提供2个选项：直接问 / 先看已沉淀问题
```

### 4. 每日学习单（学习模式必须）
```
□ 读取 docs/interview_4week_plan.md
□ 根据当前日期、上次学习进度、docs/reminder_config.md 的复习队列、docs/question_bank.md 的近期新问题，生成“今日学习单”
□ 开场先告诉用户：
   - 今天学什么新知识
   - 今天复习什么旧知识
   - 今天先抽哪 1-2 个“自己问过的问题”热身
   - 每部分预计花多久
   - 若今日时间不足，优先保留哪一部分
```

**今日学习单输出格式（默认）**
```markdown
今天建议这样学（总计约 120 分钟）：
1. 复习（20 分钟）
   - XX
   - 问题热身：Qxxx / Qyyy
2. 新知识（80 分钟）
   - XX
3. 收束（20 分钟）
   - XX

如果今天时间不够：
- 优先完成 XX
```

**生成规则**
```python
daily_plan = {
    "review": 来自 reminder_config 的 P1/P2 + 昨日薄弱点 + question_bank 近24小时新收录问题,
    "new_topic": 来自 interview_4week_plan 的当天主线任务,
    "wrap_up": 口述总结 / 错题 / 费曼输出,
    "time_budget": 默认 20/80/20 分钟
}

IF 用户明确说"先学新知识":
    仍然先展示今日学习单
    但执行顺序改为 新知识 -> 复习 -> 收束

IF 用户时间明显不足（<60 分钟）:
    压缩为 10/40/10 分钟
    只保留 1 个新知识点 + 1 个最高优先级复习点

IF 当前为提问模式:
    不生成完整“今日学习单”
    改为输出“本次提问单”：
      - 当前问题
      - 预期归类模块
      - 是否进入复习
```

## 会话进行中流程

### 提问识别与记录（关键改进）

**模式总原则**
```python
IF 当前为学习模式:
    保留现有 [记录候选] 机制
    高价值问题在会话结束时决定是否写入 question_bank.md

IF 当前为提问模式:
    默认把问题视为待沉淀资产
    每题答完立刻执行：直接答案 -> 一句话总结 -> 模块/知识点归类 -> 是否加入复习
```

**核心原则：默认偏向记录，避免漏记**
```python
# 判断目标不是"严格筛掉无关问题"，而是"尽量不漏掉暴露疑惑的学习问题"
IF 无法快速确定"这只是随口确认"还是"真实认知卡点":
    先标记为 [记录候选]
    会话结束时再决定是否正式写入 question_bank.md / question_analysis.md
```

**触发条件（满足任一即记录候选）：**
```python
IF 用户问题包含以下特征:
    - 以"为什么"、"怎么"、"如果"开头
    - 以"是不是"、"相当于"、"能不能理解为"、"可不可以认为"开头
    - 明确表达困惑、犹豫、拿不准
    - 对我刚讲过的内容提出二次确认或换一种说法确认
    - 涉及概念边界、对比、实现细节
    - 追问原理或设计决策
    - 提出反例或变体
    - 要求澄清术语或参照系
    - 试图把两个相近概念对齐，判断"是不是一回事"
    - 追问公式中某一项、某一步推导、某个张量维度的含义
    - 询问某种做法"省的到底是什么"、"影响的是哪一项成本"
    - 询问某个机制在特殊场景、边界条件、失败场景下是否还成立
THEN:
    标记为待记录问题
    立即在上下文中保存（草稿）
```

**高优先级信号（即使语气很随意，也应记录候选）：**
```python
IF 用户问题体现以下任一模式:
    - "我有点没想通/我有点疑惑/这里有点绕"
    - "是不是可以这样理解..."
    - "那为什么前面说X，这里又是Y？"
    - "这里和XX有什么区别？"
    - "如果换成XX会怎样？"
    - "这个结论是不是只在XX条件下成立？"
THEN:
    优先标记为 [记录候选]
```

**不宜漏记的隐性提问类型：**
```python
IF 用户没有用问号，但本质上在暴露认知冲突:
    - "我理解成A，但你好像在说B"
    - "感觉这里和前面的结论对不上"
    - "这个地方我总容易混"
    - "我知道结论，但不知道为什么"
THEN:
    仍按提问处理并记录候选
```

**可以只做 [快速回答] 的情况（满足多数特征才可不记）：**
```python
IF 同时满足:
    - 只是流程性确认，不暴露知识盲区
    - 不涉及原理/边界/对比/实现
    - 不影响后续学习路径
    - 不是本模块核心知识点
THEN:
    可标记为 [快速回答]
ELSE:
    默认使用 [记录候选]
```

**实时标记（用户可见）：**
```
用户提问后，我的响应开头应包含：
"[记录候选]" 或 "[快速回答]"

示例：
用户："Flash Attention的重计算是哪部分？"
我："[记录候选] 重计算的是Attention矩阵S = QK^T..."
```

**实时草稿最少要保存的信息：**
```python
draft = {
    "question": 用户原话或尽量贴近原话的转述,
    "module": 当前模块,
    "trigger": 触发原因,   # 如: 概念边界 / 二次确认 / 设计决策 / 隐性困惑
    "suspected_gap": 初步判断的盲区类型
}
```

**会话结束前的二次扫描（防漏记）：**
```python
FOR 本次会话中的每一次用户追问/澄清/反问:
    IF 当时未标 [记录候选]:
        再检查是否满足以下任一条件:
            - 暴露了"混淆/犹豫/边界不清"
            - 是对核心概念的二次确认
            - 让我补充了对比、图示、公式、例外情况
        IF 是:
            补入 session_questions
```

**question bank 写入判定**
```python
IF 当前为提问模式 AND 问题属于LLM学习范围:
    默认写入 docs/question_bank.md

ELSE IF 当前为学习模式:
    IF 问题满足以下任一条件:
        - 属于当前模块核心知识点
        - 暴露稳定盲区或重复混淆
        - 后续值得单独复习/抽查
        - 具有较高迁移价值（边界、对比、设计动机、工程权衡）
    THEN:
        写入 docs/question_bank.md
    ELSE:
        仅保留在会话层或 docs/question_analysis.md
```

**提问模式标准输出模板**
```markdown
问题：...
回答：...
总结：...
归类：Mxx / 知识点
复习状态：新收录 / 会但不稳 / 不加入复习（仅流程确认）
```

## 会话结束流程（关键！防止遗忘）

### 强制检查清单（必须在说"再见"前完成）

```markdown
## 会话关闭检查清单

### 文档更新（必须全部完成）
□ 1. 更新 docs/progress.md
   - 本次学习主题和时长
   - 新完成的内容标记为 ✅
   - 待办事项更新

□ 2. 更新 docs/reminder_config.md
   - 上次学习日期改为今天
   - 下次学习建议更新
   - 复习检测题更新

□ 3. 更新 docs/question_bank.md（新增，默认先检查）
   - 提问模式问题是否已写入
   - 学习模式高价值问题是否已转入知识库
   - 是否补充模块/知识点/复习状态/下次复习建议

□ 4. 更新 docs/question_analysis.md（最容易遗忘！）
   - 检查本次会话是否有标记"[记录候选]"的提问
   - 为每个提问添加分析块
   - 更新提问画像统计
   - 如果提问数≥3，更新思维特点雷达图

□ 5. 对照 docs/interview_4week_plan.md
   - 标记今日任务是否完成 / 顺延 / 偏离
   - 若偏离，补一句“明天如何接回主线”

### 状态同步（口头向用户确认）
□ 向用户宣读："本次已记录X个提问，其中Y个进入 question bank，更新了Z个文件"
□ 确认下次学习目标
□ 设置提醒（如果需要）
```

## 防遗忘的保险机制

### 保险1: 提问计数器
```python
# 在会话中维护计数
session_questions = []

def on_user_question(question):
    if is_llm_related(question):
        session_questions.append({
            "content": question,
            "timestamp": now(),
            "draft_analysis": quick_analyze(question),
            "bank_candidate": should_enter_question_bank(question)
        })
        return f"[Q{len(session_questions)}] {response}"
```

### 保险2: 结束会话时强制盘点
```
用户说"下次继续"、"先这样"、"结束"等时：
→ 触发会话结束流程
→ 必须显示检查清单
→ 用户确认"更新完成"后才真正结束
```

### 保险3: 文件写入确认
```
每次Write/Edit后：
→ 立即Read该文件确认写入成功
→ 向用户报告："已更新X文件，新增Y内容"
```

## 用户监督机制

### 用户可以触发强制更新
```
用户说：
- "记录这个问题" → 立即写入question_analysis.md
- "加入知识库" → 立即写入question_bank.md
- "更新进度" → 立即更新progress.md
- "检查清单" → 显示当前会话的检查清单状态
- "你忘了记录" → 我立即道歉+补录+解释防遗忘措施
```

### 透明化展示
```
每次更新后向用户展示：
"✅ 已更新 docs/question_bank.md
   - 新增 Q0xx: xxx
   - 归类到 Mxx / 知识点
✅ 已更新 docs/question_analysis.md
   - 补充认知模式分析"
```

## 错误处理流程

### 如果用户发现我忘了记录
```
1. 立即道歉
2. 询问："请指出是哪个问题，我立即补录"
3. 补录到question_bank.md / question_analysis.md
4. 在文档中添加：
   "补录说明: 此问题原本在X会话提出，因Y原因漏记，现已补录"
5. 报告："已补录，并更新了防遗忘流程以避免再次发生"
```

## 本次改进的验证

### 测试新的SOP
```
下次会话应验证：
□ 开始时是否正确读取了核心文件（含 question_bank.md）

□ 提问后是否有"[记录候选]"标记
□ 提问模式下是否产出“回答 -> 总结 -> 归类 -> 回写”
```

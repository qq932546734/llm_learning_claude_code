# 学习会话标准操作流程 (SOP)

> 确保学习连续性、记录完整性的检查清单

---

## 会话开始流程

### 1. 恢复上下文（必须）
```
□ 读取 docs/progress.md
□ 读取 docs/reminder_config.md
□ 读取 docs/question_analysis.md（最后部分）
□ 读取 docs/taxonomy.md（模块编号口径）
□ 检查上次学习日期，判断是否需要复习检测
```

### 2. 状态确认
```
□ 向用户确认当前状态："上次我们学到XX，今天继续？"
□ 列出待办事项（如有）："你还有X个费曼输出/练习题待完成"
□ 提供3个选项：继续/复习/新主题
```

---

## 会话进行中流程

### 提问识别与记录（关键改进）

**触发条件（满足任一即记录）：**
```python
IF 用户问题包含以下特征:
    - 以"为什么"、"怎么"、"如果"开头
    - 涉及概念边界、对比、实现细节
    - 追问原理或设计决策
    - 提出反例或变体
    - 要求澄清术语或参照系
THEN:
    标记为待记录问题
    立即在上下文中保存（草稿）
```

**实时标记（用户可见）：**
```
用户提问后，我的响应开头应包含：
"[记录候选]" 或 "[快速回答]"

示例：
用户："Flash Attention的重计算是哪部分？"
我："[记录候选] 重计算的是Attention矩阵S = QK^T..."
```

## 会话结束流程（关键！防止遗忘）

### 强制检查清单（必须在说"再见"前完成）

```markdown
## 会话关闭检查清单

### 文档更新（必须全部完成）
□ 1. 更新 docs/progress.md
   - 本次学习主题和时长
   - 新完成的内容标记为 ✅
   - 待办事项更新

□ 2. 更新 docs/reminder_config.md
   - 上次学习日期改为今天
   - 下次学习建议更新
   - 复习检测题更新

□ 3. 更新 docs/question_analysis.md（最容易遗忘！）
   - 检查本次会话是否有标记"[记录候选]"的提问
   - 为每个提问添加分析块
   - 更新提问画像统计
   - 如果提问数≥3，更新思维特点雷达图

### 状态同步（口头向用户确认）
□ 向用户宣读："本次已记录X个提问，更新Y个文件"
□ 确认下次学习目标
□ 设置提醒（如果需要）
```

### 保险2: 结束会话时强制盘点
```

→ 触发会话结束流程
→ 必须显示检查清单

```

## 用户监督机制

### 用户可以触发强制更新
```
用户说：
- "记录这个问题" → 立即写入question_analysis.md
- "更新进度" → 立即更新progress.md
- "检查清单" → 显示当前会话的检查清单状态
- "你忘了记录" → 我立即道歉+补录+解释防遗忘措施
```

### 透明化展示
```
每次更新后向用户展示：
"✅ 已更新 docs/question_analysis.md
   - 新增 Q006: Flash Attention重计算
   - 更新思维画像: 工程思维 50%→60%"
```

## 错误处理流程

### 如果用户发现我忘了记录
```
1. 立即道歉
2. 询问："请指出是哪个问题，我立即补录"
3. 补录到question_analysis.md
4. 在文档中添加：
   "补录说明: 此问题原本在X会话提出，因Y原因漏记，现已补录"
5. 报告："已补录，并更新了防遗忘流程以避免再次发生"
```

# 知识点划分与命名规范（统一口径）

> 目的：解决 README / curriculum / progress 中“模块编号”口径不一致导致的歧义。

## 1) 统一规则（从现在开始默认使用）

### 1.1 唯一的“课程模块（最外围）”编号：`M01` ~ `M16`
- **`Mxx` 只表示 README 里的 16 个模块**（最外围知识点）。
- 以后对话里你说的“`M3` / `M03`”，默认就是 **`M03: 分词与表示`**。

### 1.2 模块内子主题编号：`M03-01` / `M03-02` …
- 例：`M03-01 Tokenization`，`M03-02 Embedding`
- 例：`M10-03 Speculative Decoding`

### 1.3 其它文档的编号不再用 `Mxx`
为避免冲突，两个“非 canonical 大纲”使用不同前缀：
- `Axx`：`curriculum/modules_outline.md` 的 **8 个宏模块**（Macro Outline）
- `Sxx`：`curriculum/syllabus.md` 的 **16 个学习路径模块**（Syllabus Track）

> 结论：**`Mxx` 只有一套**；`Axx`、`Sxx` 只是“另一种组织方式”，不与 `Mxx` 抢编号。

### 1.4 模块内知识点标签规范
每个模块内的知识点统一使用以下标签，便于决定讲解深度与复习优先级：
- `核心必会`：不掌握会直接影响后续模块理解。
- `面试高频`：面试中高概率直接提问或变体追问。
- `工程重点`：与系统实现、性能、资源估算或调参直接相关。
- `扩展阅读`：不影响主线推进，但有助于拓展视野或研究深度。

### 1.5 模块收录下限
每个 `Mxx` 模块至少包含：
- 1 组概念与原理的最小闭环。
- 1 组易错点或边界条件。
- 1 个工程联系（估算题、实现题、设计题三选一）。
- 1 个面试高频问法与标准回答骨架。
- 明确的前置依赖与后续关联。

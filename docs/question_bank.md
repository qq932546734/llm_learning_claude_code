# 问题知识库（Question Bank）

> 作用：沉淀可复习的高价值问题，按知识点聚合，而不是按时间散落。
> v1 原则：仅收录 LLM 学习相关问题；`提问模式` 默认写入，`学习模式` 仅高价值问题写入。

## 使用规则
- 开场可进入 `学习模式` 或 `提问模式`。
- `学习模式`：提问继续按 `[记录候选]` 机制处理；会话结束时再判断是否进入知识库。
- `提问模式`：默认执行“回答 -> 总结 -> 归类 -> 回写 -> 是否加入复习”。
- 本文档负责“问题沉淀与复习入口”，不替代 `docs/question_analysis.md` 的认知画像功能。

## 记录模板

```yaml
问题ID: Qxxx
日期: YYYY-MM-DD
模块: Mxx
知识点: Xxx
问题: ...
总结: ...
状态: 新收录 / 会但不稳 / 已掌握
复习建议: 24小时内 / 3天内 / 7天内 / 暂不安排
来源模式: 学习模式 / 提问模式
```

## 复习使用方式
- 每次学习开场，优先从最近 24 小时的新收录问题中抽 1-2 个做热身。
- 同一知识点下出现多个问题时，尽量聚合到同一小节，避免重复散落。
- 若某问题连续两次复习都能稳定口述，可将状态从 `新收录` 调整为 `已掌握`。

## M03 分词与表示

### Q030

```yaml
问题ID: Q030
日期: 2026-03-18
模块: M03
知识点: WordPiece / Unigram / tokenizer 的层级关系
问题: "这个问题本身是不是不大对？WordPiece、unigram不应该是跟BPE并列的吗？怎么跟tokenizer并列了"
总结: tokenizer 是上位概念，BPE / WordPiece / Unigram 是 tokenizer 的几种训练或构词方法，三者彼此并列，但不与 tokenizer 并列。
状态: 会但不稳
复习建议: 3天内
来源模式: 学习模式
```

### Q031

```yaml
问题ID: Q031
日期: 2026-03-18
模块: M03
知识点: WordPiece vs Unigram 的概率口径
问题: "为什么说 WordPiece 是‘似然导向的训练准则’，而 Unigram 更像‘显式概率模型’？"
总结: WordPiece 更强调用似然收益指导 merge，是训练准则层面的近似贪心构词；Unigram 显式给 token 和切分分配概率，一句话可有多种切分且各自有概率，因此更像显式概率模型。
状态: 会但不稳
复习建议: 24小时内
来源模式: 学习模式
```

## M10 解码策略

### Q032

```yaml
问题ID: Q032
日期: 2026-03-18
模块: M10
知识点: 开放式生成中的 beam search 边界
问题: "为什么 beam search 在很多开放式生成任务里不常作为默认选项？"
总结: 核心原因不是先看成本，而是 beam search 优化高概率序列，和开放式生成追求自然、多样、少模板化的任务目标有错位；成本更高只是附加原因。
状态: 会但不稳
复习建议: 3天内
来源模式: 学习模式
```

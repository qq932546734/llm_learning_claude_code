# 详细教学指南

## 1. 诊断问题设计原则

每个模块的前置诊断包含 5 个问题，按以下分布：

| 题号 | 目的 | 示例 |
|------|------|------|
| Q1 | 探测基础概念掌握 | "Self-Attention 的核心公式是什么？" |
| Q2 | 探测数学理解深度 | "为什么除以 sqrt(d_k)？如果不除会怎样？" |
| Q3 | 探测工程实现经验 | "你有没有实现过 Multi-Head Attention？" |
| Q4 | 探测实际应用经验 | "在项目中遇到过什么注意力相关的问题？" |
| Q5 | 探测拓展知识 | "你知道哪些注意力变体？它们的动机是什么？" |

### 评分标准

根据回答自动判断层级：

```
L1 (了解层): 能回答 Q1，Q2-Q5 回答模糊或不知道
L2 (理解层): 能回答 Q1-Q3，Q4-Q5 有部分了解
L3 (应用层): 能详细回答所有问题，有自己的见解
```

## 2. 各层级教学内容示例

以 **Self-Attention** 为例：

### L1 层教学 (15-20 分钟)

**目标**: 建立直观理解

**内容**:
- 一句话定义：Self-Attention 让序列中的每个位置都能"看到"其他位置
- 核心思想：Query 问，Key 答，Value 给答案
- 可视化：展示注意力热力图
- 关键公式：只给出 $Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V$
- 为什么重要：对比 RNN/CNN，解决长距离依赖

**输出要求**: 用一句话解释 Self-Attention 是什么

### L2 层教学 (30-45 分钟)

**目标**: 深入理解机制

**内容**:
- 完整公式推导：从 Scaled Dot-Product 到输出
- 维度分析：Q/K/V 的维度变化，为什么 $d_k$ 影响 Softmax
- 数学原理：为什么要除以 $\sqrt{d_k}$？Softmax 梯度饱和问题
- Mask 机制：Padding Mask + Causal Mask 详解
- 复杂度分析：$O(n^2)$ 的由来与影响
- 与 RNN/CNN 的对比：参数量、计算量、感受野

**输出要求**:
1. 推导 Scaled Dot-Product Attention
2. 解释为什么 $d_k=64$ 时除以 8

### L3 层教学 (60-90 分钟)

**目标**: 动手实现与优化

**内容**:
- 从零实现：NumPy 版 Self-Attention
- PyTorch 优化版：利用广播和矩阵运算
- 内存优化：In-place 操作、Gradient Checkpointing
- 性能分析：用 PyTorch Profiler 找出瓶颈
- 实际调试：处理数值稳定性问题 (数值溢出)
- 扩展阅读：FlashAttention 的 CUDA 优化思想

**输出要求**:
1. 实现一个高效的 Multi-Head Attention 模块
2. 与 PyTorch nn.MultiheadAttention 对比性能
3. 实现一个简单的 Causal LM 并训练

## 4. 刻意练习设计

### 练习类型

| 类型 | 目的 | 示例 |
|------|------|------|
| 概念辨析 | 区分易混淆概念 | "Attention vs Self-Attention vs Cross-Attention" |
| 数学推导 | 巩固理论基础 | "推导 Multi-Head Attention 的输出维度" |
| 代码填空 | 检验实现细节 | "补全 Attention 中的 Mask 逻辑" |
| Bug 修复 | 培养调试能力 | "这个 Attention 实现为什么产生 NaN？" |
| 开放设计 | 综合运用 | "设计一个处理超长文本的 Attention 机制" |

### 难度分级

- ⭐ 基础：直接应用刚学的知识
- ⭐⭐ 进阶：需要结合多个知识点
- ⭐⭐⭐ 挑战：需要查阅资料或创新

## 5. 实时反馈机制

作为你的 AI 导师，我会：

1. **即时判断**: 从你的回答中判断理解程度
2. **动态调整**: 如果发现你 L2 的内容已经很熟，直接跳到 L3
3. **追问引导**: 用苏格拉底式提问引导你发现问题
4. **错误分析**: 不只是告诉你对错，而是分析错误原因
5. **正向强化**: 指出你的进步，增强学习动力

## 6. 学习节奏建议

| 场景 | 建议节奏 |
|------|----------|
| 工作日晚上 | 1 个小节 (L1 20分钟 / L2 40分钟) |
| 周末集中 | 1 个完整模块 (含练习和项目) |
| 假期冲刺 | 1 个 Phase (4-6 个模块) |

## 7. 学习质量检查点

每个 Phase 结束时进行：

1. **概念测试**: 20 道选择题，80% 通过
2. **代码考核**: 实现一个核心算法
3. **论文阅读**: 精读 1 篇经典/前沿论文
4. **项目实战**: 完成一个综合项目
5. **教学演练**: 向我（AI导师）讲解一个概念

通过全部检查点后，才能进入下一阶段。

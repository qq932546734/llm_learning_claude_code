## 风险等级定义
- `新学`：刚学完，需要在 24 小时内回顾。
- `会但不稳`：理解已建立，但容易在细节或口述中失真。
- `已掌握`：当前能独立解释，可拉长回访周期。
- `待回访`：曾学过但近期未复述，需抽查防遗忘。

## 复习调度规则
- `新学`：24 小时回顾。
- `会但不稳`：3 天复测。
- `已掌握`：7-14 天抽查。
- 做错过或提问暴露盲区：隔天必复习，并进入错题池。

## 学习单合并规则（v2）
- `interview_4week_plan.md` 决定当天必须推进的主线任务；若上一计划日未完成，先补最近未完成日，不跳过主线。
- `reminder_config.md` 决定当天到期 / 逾期的复习任务；复习必须按“上次学习 / 上次复习日期 + 间隔规则”触发，不能因为切换主线而丢掉。
- `progress.md` 只负责补充“未收束项 / 本周薄弱点 / 与当天主线强相关的迁移题”，不能单独替代当天主线。
- 开场生成“今日学习单”时，先列 `必做`：当天主线 + 已逾期 / 今天到期的复习；再列 `应做`：强相关承接题；最后列 `可选`：额外迁移题或收束题。
- 若用户明确给出可用时间，可以压缩题量和深度；若用户未说明时长，不再默认写死 `120 分钟`，而是保持优先级顺序执行。
- 课程补强类新主题（`E01` ~ `E06`）默认不插入当前进行中的主线周；若用户允许延长学习周期，则统一放入 `Week 4+` 延长期或 `Day 7` 弹性日。
- 补强主题在“正式开始学习”之前，不提前生成新的 `24 小时 / 3 天` 复习任务，避免打乱既有复习节奏。
- 已学模块的静默回写默认视为“课程资产整理”，不因为正文变深而重新生成该模块的 `24 小时 / 3 天` 复习。
- 若静默回写中包含学习者此前未正式学过、且确属高频面试点的“净新增知识点”，需显式标记后再进入复习队列；进入方式应为 `1 个判断句 + 1 个 trade-off + 1 道小题` 的补丁式回访，而不是整模块重开。

## 问题沉淀复习规则（v1）
- `提问模式` 新收录的问题：默认进入复习队列。
- `学习模式` 下的高价值问题：会话结束时决定是否进入复习队列。
- 属于当前模块核心知识点的问题：默认 `P1`。
- 属于扩展理解、迁移思考、旁支主题的问题：按 `P2/P3` 排队。
- 每次开场生成“今日学习单”时，优先抽 1-2 个你自己问过的问题做热身复习。
- 若某问题标记为 `复习建议: 暂不安排`，即使是最近新收录，也不作为热身抽查候选。

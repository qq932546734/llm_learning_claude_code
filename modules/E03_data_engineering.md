# E03: 数据工程（Data Engineering）

## 为什么学
- 训练效果的上限经常先被数据质量和数据配比卡住，而不是先被模型结构卡住。
- 这一块能把“模型为什么学歪”“为什么 SFT / 偏好优化效果不稳”拉回到数据层看。

## 前置依赖
- M02 预训练理论与方法
- M03 分词与表示
- M07 指令微调

## 诊断题
1. 为什么“数据更多”不一定比“数据更干净”更重要？
2. 去重为什么会直接影响 benchmark 表现的可信度？
3. 多数据源混合时，为什么需要配比和课程学习直觉？

## 核心知识点
- `核心必会`：数据工程至少包括收集、清洗、去重、质量筛选、配比和切分。
- `核心必会`：去重不只是省存储，它还影响记忆污染、benchmark 泄漏和训练稳定性。
- `核心必会`：多来源数据混合时，真正要调的是“信号强度、覆盖范围、任务权重”，不是只拼总 token 数。
- `工程重点`：课程学习、样本重采样、source weighting 常用于避免模型过早被噪声或单一任务分布带偏。
- `面试高频`：讨论数据问题时，优先按“质量 -> 多样性 -> 配比 -> 污染控制”展开。

## 易错点 / 边界条件
- 容易把数据工程理解成纯 ETL。
- 容易把“清洗得更狠”误当成必然更好，忽略多样性损失。
- 容易忽略训练/验证/评测集之间的污染链路。

## 面试高频问法
### 问法 1
问题：为什么数据工程会直接影响模型能力上限？
- 一句话答案：因为模型学到的分布、覆盖范围和噪声边界都由数据决定，训练只是在这些约束内优化。
- 展开框架：覆盖 -> 噪声 -> 配比 -> 污染 -> 行为后果。

### 问法 2
问题：去重为什么重要？
- 一句话答案：它不只减少重复样本，还影响数据利用率、记忆污染、评测可信度和训练信号质量。
- 展开框架：重复样本 -> 过拟合/污染 -> benchmark 泄漏 -> 何时不能过度去重。

## 完成标准
- 诊断题正确率达到 80%。
- 能解释清洗、去重、配比三者的作用边界。
- 完成 1 道数据流水线或配比题。

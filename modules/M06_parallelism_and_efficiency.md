# M06: 模型并行与效率

## 为什么学
- 大模型训练和部署离不开资源切分；这是工程与面试都非常高频的模块。
- 这一块一旦掌握，会明显提升你对训练成本和架构选型的判断力。

## 前置依赖
- M02 预训练优化基础
- 参数、梯度、优化器状态、activation 的概念

## 诊断题
1. ZeRO 1/2/3 分别分片什么？为什么 Stage 1 性价比常最高？
2. All-Reduce 在 DDP 中扮演什么角色？通信量与什么成正比？
3. 训练显存主要被哪些项占用（参数/梯度/优化器状态/activation）？

## 核心知识点
- `核心必会`：数据并行 / 张量并行 / 流水线并行的切分维度和通信模式。
- `核心必会`：ZeRO Stage 1/2/3 逐步分片优化器状态、梯度、参数。
- `工程重点`：FSDP 与 ZeRO 的目标相近，但模块粒度和工程路径不同。

## 易错点 / 边界条件
- 容易只背 Stage 1/2/3，而说不清“分片对象”和“为什么逐步变贵”。
- 容易把 FSDP 和 ZeRO 说成完全一样。
- 容易只从显存看收益，忽略通信开销和工程复杂度。

## 面试高频问法
### 问法 1
问题：ZeRO Stage 1/2/3 的区别是什么？
- 一句话答案：它们按递进方式分片优化器状态、梯度、参数，显存省得更多，但通信和实现复杂度也更高。
- 展开框架：分片对象 -> 节省哪部分显存 -> 新增什么通信成本。
- 常见追问点：为什么 Stage 1 性价比高？什么时候直接上 Stage 3？

### 问法 2
问题：FSDP 和 ZeRO 的主要差别是什么？
- 一句话答案：目标相似，都是做全分片，但 FSDP 更强调模块级参数 gather / shard，工程接口与生态位置不同。
- 展开框架：目标 -> 分片时机 -> 参数 gather 方式 -> 使用场景。
- 常见追问点：FSDP 更适合什么框架？和 tensor parallel 如何组合？

- 面试里很常用 RLHF / PPO / DPO 来区分概念理解和工程理解。

## 诊断题
1. PPO 为什么常被描述为“四模型协作”？哪些冻结、哪些训练？
2. DPO 为什么不需要显式 Reward Model？隐式奖励形式是什么？
3. DPO 里 `beta` 变大更偏“激进”还是“保守”？为什么？

## 核心知识点
- `核心必会`：RLHF 三阶段 SFT -> RM -> PPO / DPO。
- `核心必会`：Bradley-Terry 偏好建模、Advantage、clipped objective。
- `工程重点`：PPO 显存与训练复杂度高；DPO 更简单但有长度偏置、beta 稳定性问题。
- `面试高频`：reward vs margin、隐式奖励、DPO 与 PPO 的取舍。

## 易错点 / 边界条件
- 容易把 DPO 的 reward 和 margin 当成同一个量。
- 容易知道 beta 影响强度，却说不清它和 reference 约束的关系。
- 容易只说 DPO 简单，却忽略长度偏置和数据构造的重要性。

## 面试高频问法
### 问法 1
问题：PPO 和 DPO 的主要区别是什么？
- 一句话答案：PPO 显式做强化学习更新，DPO 直接在偏好对上做闭式形式的策略优化。
- 展开框架：训练对象 -> 需要哪些模型 -> 资源成本 -> 稳定性和工程复杂度。
- 常见追问点：为什么 DPO 更省显存？它真的完全替代 PPO 吗？

### 问法 2
问题：DPO 为什么不需要 Reward Model？
- 一句话答案：因为最优策略在偏好目标下可写成相对 reference policy 的闭式形式，奖励被隐式吸收进 log-ratio。
- 展开框架：偏好学习目标 -> 闭式解洞察 -> 隐式奖励 -> 实际训练形式。
- 常见追问点：beta 的作用是什么？reward 和 margin 有什么区别？

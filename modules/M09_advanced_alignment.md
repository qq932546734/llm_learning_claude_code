# M09: 高级对齐技术

## 为什么学
- 这是从“能对齐”走向“理解对齐边界与替代路径”的模块。
- 这一块的价值不在于多记几个缩写，而在于把“反馈来源、原则表达方式、人类校准”三层关系压成稳定口径。

## 前置依赖
- M08 RLHF
- 基本安全与偏好优化直觉

## 诊断题
1. RLAIF 相比 RLHF 的关键差异是什么？它解决了什么成本问题？
2. Constitutional AI 和一般的 RLAIF 有什么关系与区别？
3. 为什么 `helpful / harmless / honest` 往往有张力，而不是一个目标自然兼顾？
4. 为什么更强的 AI judge 仍不能完全替代人类校准？

## 核心知识点
- `核心必会`：`SFT` 更像在教模型“怎么答”，偏好对齐更像在多个可答方案里学“哪个更好”。
- `核心必会`：`RLAIF` 用 `AI feedback` 替代部分人类反馈，主要解决的是标注成本与扩展性问题，不是自动获得“客观标准”。
- `核心必会`：`Constitutional AI` 可理解为“显式原则 + 自我批评/修正”的 `AI feedback` 路线，重点是更显式、更可审查，而不是天然更客观。
- `工程重点`：高级对齐常见失败模式包括奖励黑客、sycophancy、过度拒答、目标错配与 judge 偏差放大。
- `面试高频`：解释 `RLHF / RLAIF / Constitutional AI` 时，优先按“反馈来源 -> 原则表达方式 -> 是否仍需人类校准”三层展开。

## 易错点 / 边界条件
- 容易把“符合偏好”误解为“真正对齐”，忽略偏好标签本身就带价值判断。
- 容易把“显式原则”误解成“更客观”，但原则如何写、冲突时谁优先仍是人定的。
- 容易把“AI judge 更强”误解成“可以完全不要人”，却忽略 judge 会继承并放大既有偏差。
- 容易只记住 `helpful vs harmless`，漏掉 `honest` 和“让用户满意”并不等价。

## 面试高频问法
### 问法 1
问题：`RLHF / RLAIF / Constitutional AI` 的关系是什么？
- 一句话答案：三者都在做偏好对齐，只是反馈来源和原则表达方式不同，而人类校准始终没有真正消失。
- 展开框架：共同目标 -> `RLHF` 的人类反馈 -> `RLAIF` 的 `AI feedback` -> `Constitutional AI` 的显式原则 -> 人类校准为何仍在。
- 常见追问点：`Constitutional AI` 和普通 `RLAIF` 是不是一回事？为什么更显式不等于更客观？

### 问法 2
问题：为什么 `helpful / harmless / honest` 往往存在张力？
- 一句话答案：因为真实任务里“更有帮助”“更安全”“更诚实”常会把模型拉向不同方向，不存在一个单一标量能无损兼顾。
- 展开框架：典型冲突场景 -> `honest` 不等于让用户满意 -> 规则化反馈为什么更易推向保守 -> 工程上如何取舍。
- 常见追问点：为什么规则化 `AI feedback` 容易让模型更保守？过度拒答算哪种失败？

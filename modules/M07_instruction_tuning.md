## 诊断题
1. 为什么少量高质量指令数据也能显著改变助手行为？
2. SFT 容易导致哪些副作用（遗忘、风格漂移、过拟合）？
3. LoRA 为什么能省显存？它限制了什么表达能力？

## 核心知识点
- `核心必会`：SFT 调整的是输出行为分布，而非彻底重建知识。
- `核心必会`：数据格式、多样性和难度覆盖比“量大”更重要。
- `工程重点`：LoRA / QLoRA / DoRA 是不同的参数高效微调路径。

## 易错点 / 边界条件
- 容易把“指令遵循变好了”误解为“知识更多了”。
- 容易忽略数据分布导致的风格偏置和遗忘。

#### 本次学习总结（2026-03-12）

**1 页总结**

- SFT 的核心作用不是“给模型新增很多知识”，而是把预训练模型已有能力重新对齐到“用户指令 -> 助手回答”的输出分布上。
- 少量高质量指令数据也能显著改变助手行为，因为监督信号更贴近目标行为：它直接教模型如何遵循指令、控制语气、组织回答结构和遵守格式。
- SFT 常见副作用包括：遗忘部分预训练阶段的泛化能力、风格偏置、模板过拟合，以及数据分布过窄时在其他任务上的能力退化。
- LoRA 的直觉是：冻结原始大权重，只学习一个低秩增量 `ΔW = AB`，从而显著减少可训练参数、梯度和优化器状态的显存占用。
- LoRA 的边界是：它把权重更新限制在低秩子空间中；如果任务需要非常复杂或接近全秩的改动，LoRA 可能不够灵活。
- 高质量 SFT 数据不一定很多，但必须高信号密度：真实任务分布、事实约束、边界清晰、回答可执行、覆盖负面情绪和拒答场景。

**错题 / 易混点**

- 不要把“指令遵循变好了”误解为“模型知识明显增加了”；SFT 更主要是改变行为分布。
- 不要把 LoRA 说成“把原矩阵变成两个小矩阵”；更准确地说，是冻结原矩阵，只训练低秩增量。
- 不要只说 SFT 会“改变预测能力”；更具体的副作用是遗忘、风格偏置、模板过拟合和分布外退化。

**练习产物：客服助手 SFT 数据样例原型**

1. 职责内正常解答：听单冷区导致长时间无单，基于已知热区信息给出解释与建议。
2. 职责内查事实后回答：查询奖励规则与当前完单进度，再告知差多少单可达标。
3. 职责外礼貌转接：失物招领问题不在职责范围内，说明边界并引导联系人工客服。
4. 职责内信息不足先澄清：奖励未到账时，不编原因，先确认奖励类型与完成时间。
5. 职责内负面情绪处理：司机强烈抱怨派单，先承接情绪，再基于听单设置与订单事实解释原因并给出可执行建议。

## 费曼输出
- 用 120 字解释：SFT 改变的到底是什么？

### 本次费曼输出（2026-03-12）
SFT 主要改变的是模型的输出行为分布，而不是简单给模型新增知识。它会让模型更会遵循指令，在语言风格、回答结构、格式约束和助手化交互方式上更贴近训练数据。因为预训练模型本身已经具备很多知识，SFT 更多是在把这些已有能力重新组织成“用户提问 -> 助手回答”的形式。与此同时，如果 SFT 数据分布过窄，也可能带来遗忘、风格偏置、模板过拟合，以及在其他任务上的能力退化。

# M15: 模型可解释性

## 为什么学
- 它帮助你分辨“看起来像解释”和“真正可信的解释”。
- 这块的核心不是会画图，而是知道哪些方法只是在做相关性可视化，哪些方法更接近因果证据。

## 前置依赖
- M01 模型结构
- 基本统计与实验设计直觉

## 诊断题
1. 为什么“注意力权重”不一定等于“解释”？
2. 探测分类器（probe）常见的泄漏 / 虚假相关风险是什么？
3. 如果一个 probe 准确率很高，为什么仍不能直接得出“模型显式存了这个概念”？
4. 为什么解释性研究往往离不开对照实验或干预实验？

## 核心知识点
- `核心必会`：attention map、probe、特征可视化、representation analysis 都很有用，但大多先给的是诊断线索，不是自动成立的解释结论。
- `核心必会`：高准确率 probe 可能只是利用了表示中的可线性读取信息，不能直接证明该概念就是模型做决策时的因果中介。
- `核心必会`：相关性可视化与因果解释要区分；若想更接近因果，需要干预、消融、activation patching 等实验。
- `工程重点`：解释性方法常在“便宜但不够可信”和“更可信但更重实验成本”之间权衡。
- `面试高频`：优先用“观察 -> 对照 -> 干预”三层框架，说明为什么仅凭热力图或 probe 结果不能下结论。

## 易错点 / 边界条件
- 容易把“能画图”误当成“能解释模型行为”。
- 容易忽略 probe 本身带来的表达能力。
- 容易把“可线性读出”误当成“模型实际用来决策”。
- 容易在没有对照组时，过度解读某个可视化模式。

## 面试高频问法
### 问法 1
问题：为什么说 attention 不等于 explanation？
- 一句话答案：因为注意力权重展示的是相关分配模式，不足以单独证明这些位置就是模型最终决策的因果依据。
- 展开框架：权重可视化能说明什么 -> 为什么不足 -> 需要什么额外证据 -> 干预实验的作用。
- 常见追问点：那 attention map 还有没有价值？什么时候它仍然有用？

### 问法 2
问题：probe 高准确率说明了什么，又说明不了什么？
- 一句话答案：说明表示里可能包含可读出的信息，但说明不了该信息是否是模型真正依赖的决策机制。
- 展开框架：可读出性 -> probe 泄漏风险 -> 对照/随机标签/层间比较 -> 何时再上干预实验。
- 常见追问点：为什么 probe 会“学会任务”？怎么设计更稳的 probe 实验？

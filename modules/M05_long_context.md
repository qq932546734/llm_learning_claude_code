# M05: 长文本建模

## 为什么学
- 长上下文是推理成本、检索替代方案和产品体验的交叉点。

## 前置依赖
- M01 位置编码
- M04 / M12 的复杂度与推理瓶颈直觉

## 诊断题
1. 长上下文场景下，限制来自“算力”还是“显存/带宽”？为什么？
2. RoPE 外推为什么会出问题？常见工程修补有哪些？
3. RAG 与长上下文各适合什么场景？

## 核心知识点
- `核心必会`：长上下文成本往往首先爆在 KV cache 和带宽上。
- `核心必会`：位置编码外推与上下文长度扩展不是同一件事。
- `工程重点`：上下文压缩、检索、分块注意力、Ring 等路线的取舍。

## 易错点 / 边界条件
- 容易把“上下文更长”直接等同于“效果更好”。
- 容易忽略长上下文问题常常先是系统瓶颈，再是算法瓶颈。
